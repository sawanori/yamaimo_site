# 山芋の多い料理店 — ウェブサイト構築プロジェクト

## プロジェクト概要

「山芋の多い料理店」（川崎店・西葛西店・3号店）のコーポレート＋ブランドサイトを、HTML/CSS/JSでゼロから構築する。

現状は owst.jp（レストランボード）のテンプレートサイトで、ブランドの世界観がまったく伝わっていない。これを、手間暇かけた人間が設計した、AIっぽさのない、クリエイティブなウェブデザインに作り替える。

## デザインリファレンス

### 参考サイト: https://www.marco.tokyo/
三茶呑場マルコ（株式会社2TAPS）のサイト。以下の構造を参考にする:
- トップページ: コンセプト → ブランド一覧（全店舗カード） → 会社概要 → 採用 → フッター
- 店舗ページ: ヒーロー → NEWS → CONCEPT → RECOMMEND（スライダー） → 料理 → ドリンク → 空間 → アクセス → フッター
- ナビ: ハンバーガーメニュー → 全店舗一覧パネル → 予約導線を常時アクセス可能に

### マルコとの差別化（重要）
- マルコ: **ダーク背景、酒場の温もり、手書きロゴPNG**
- 山芋: **白ベース、素材感・土の温もり、明るく開放的、モダンカジュアル**
- マルコ: 縦積みカード一律構成
- 山芋: **セクションごとにレイアウトが変わる、予測不能な構成**
- マルコ: 予約導線がフッター固定
- 山芋: **スティッキーCTAが文脈に応じて表示切替**

## デザイン方針

### 方向性
**現代的・カジュアル**。白ベース×写真大きめ、親しみやすさ重視。ただし「白くてキレイなだけ」ではなく、紙のような質感のテクスチャ、非対称レイアウト、大胆なタイポグラフィで「人の手が入った設計」を感じさせる。

### AIっぽさを排除する鉄則
1. **均等配置の禁止** — カードを均等グリッドに並べない。非対称、オフセット、不均一な余白
2. **テンプレ感の排除** — 同じ構造の繰り返しにしない。セクションごとにレイアウトを変える
3. **タイポグラフィの緊張感** — 70px超の大見出しと14pxの本文が同居してよい
4. **写真の動きある配置** — 枠からはみ出す、重なる、マスクされる
5. **余白の呼吸** — セクション間paddingは80px〜200pxで可変
6. **テクスチャと質感** — べた白ではなく、微細なノイズや紙のような質感をCSS背景で実現
7. **スクロール体験** — 視差効果やフェードインのタイミングで緩急をつける

### 素材
ダミーで進める。写真はすべてプレースホルダーdiv（背景色＋テキスト）で表現。あとで実写真に差し替え前提。

## 技術要件

- **HTML5 + CSS3 + Vanilla JS**（フレームワーク不使用）
- **レスポンシブ**: モバイルファースト、375px / 768px / 1024px / 1440px
- **フォント**: Google Fonts（Zen Kaku Gothic New:wght@400;500;700, Cormorant Garamond:ital,wght@0,400;0,600;1,400, Shippori Mincho:wght@400;500）。CSS内の `@import` ではなくHTMLの `<link rel="preconnect">` + `<link rel="stylesheet">` で読み込むこと（レンダリングブロッキング回避）
- **アニメーション**: CSS transition + IntersectionObserver
- **外部ライブラリ**: Swiper.js v11.x（CDN、マイナーバージョンまで固定推奨。例: `swiper@11.2.6`）のみ。CDN読み込み失敗時は横スクロールにフォールバック
- **SEO**: セマンティックHTML、meta description、構造化データ（JSON-LD `Restaurant` / `LocalBusiness` スキーマを各店舗ページに埋め込み。営業時間・住所・電話番号を含む）。`<title>` にはローカル検索クエリを含める（例: `山芋の多い料理店 川崎 | JR川崎駅徒歩5分の自然薯・山芋料理居酒屋`）
- **アクセシビリティ**: prefers-reduced-motion対応、focus-visible、aria属性
- **Google Maps**: Embed APIを使用（API Keyなしの `maps.google.com/maps?q=` パターンは非推奨。正式なEmbed APIまたはAPI Key付きを推奨するが、初期段階ではAPI Keyなしで進め、本番公開前に切替）

## 詳細仕様の参照先

**実装時は必ず以下のファイルを順番に読み込んでから着手すること。**

1. `.claude/skills/references/design-system.md` — カラー、タイポグラフィ、スペーシング、全コンポーネント仕様（CSS変数、ボタン、見出し、プレースホルダー、カード、CTA、ナビ等のCSS実装例含む）
2. `.claude/skills/references/site-structure.md` — 全4ページの完全なセクション順序、各セクションのレイアウト指定、HTML構造例、コンテンツ配置ルール
3. `.claude/skills/references/content-data.md` — 全店舗の店舗情報、メニュー全品名・価格・説明文、コンセプト文、ブランドストーリー、NEWSダミー、RECRUIT文言
4. `.claude/skills/references/interaction-spec.md` — IntersectionObserver設定、全アニメーション種別（フェードアップ/スライドイン/スケール/スタガー）のCSS＋JS実装例、パララックス、ヘッダー挙動、モバイルメニュー開閉、Swiper設定、スティッキーCTA、スムーススクロール、ローディング、レスポンシブ挙動表

## ページ別構成サマリ

### index.html（トップページ）
1. ヒーロー（100vh、ロゴ大、スクロールインジケーター）
2. CONCEPT（ブランドストーリー、テキスト中央）
3. 三種の山芋（3カラム非均等、自然薯/大和芋/長芋）
4. Our Brands（交互レイアウト、川崎→西葛西→3号店COMING SOON）
5. PHILOSOPHY（Overlap型、テキスト＋写真重なり）
6. NEWS（最新3件リスト）
7. COMPANY（Split、会社情報テーブル）
8. RECRUIT（中央寄せ＋CTA、アクセント背景）
9. フッター（ダーク反転）

### kawasaki.html / nishikasai.html（店舗ページ）
1. ヒーロー（70vh、店舗名）
2. NEWS（横スクロールカード3件）
3. CONCEPT（Split 2段構成）
4. RECOMMEND（Swiperスライダー5品）
5. FOOD（Masonry写真グリッド＋テキスト）
6. DRINK（Split）
7. FLOOR（Full-bleed写真＋2カラム）
8. ACCESS（Google Map＋店舗情報テーブル）
9. フッター
10. スティッキーCTA（電話予約＋WEB予約）

※ 西葛西店は川崎店と同テンプレートだが、CONCEPTとFLOORのレイアウトを変えてコピペ感を排除すること。詳細はsite-structure.md参照。

### newshop.html（3号店）
- 全セクションのHTMLスケルトン + COMING SOON表示
- コンテンツはすべてプレースホルダー＋ `<!-- TODO -->` コメント

## 実装の進め方

### Phase 1: 基盤
1. design-system.md を読み、style.css にCSS変数・リセット・タイポグラフィ・共通コンポーネントを実装
2. main.js にIntersectionObserver、ヘッダー挙動、スムーススクロールを実装

### Phase 2: トップページ
3. index.html を site-structure.md の SEC-01〜SEC-09 に従って構築
4. 各セクションに reveal クラスを適用しアニメーションを確認

### Phase 3: 川崎店ページ
5. kawasaki.html を site-structure.md の SHOP-01〜SHOP-09 に従って構築
6. Swiper初期化、スティッキーCTA実装

### Phase 4: 西葛西店ページ
7. nishikasai.html を川崎店をベースに、レイアウト差分を適用して構築

### Phase 5: 3号店テンプレート
8. newshop.html をスケルトンとして構築

### Phase 6: 仕上げ
9. 全ページのレスポンシブ確認（375px, 768px, 1024px, 1440px）
10. meta description、構造化データ追加
11. prefers-reduced-motion対応確認
12. README.md 作成
